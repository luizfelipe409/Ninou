REVISÃO v75.75.119 — Remoção de CSS morto (não referenciado)
===============================================================

CONTEXTO
--------
Após a consolidação de duplicatas (v75.75.118), foi feita uma
varredura para identificar seletores CSS que não correspondem a
NENHUM elemento gerado pelo app hoje (nem em index.html, nem em
qualquer template gerado pelos módulos de js/).

METODOLOGIA
-----------
1. Foi montado um "corpus de tokens usados": todo identificador
   (palavra) que aparece em index.html, app.js, sw.js e todos os
   arquivos de js/**/*.js — incluindo os que geram HTML dinamicamente
   via template string (ex.: js/app.legacy.js, js/ui/*.js). Também
   foram mapeados usos de `dataset.algumaCoisa` para o atributo
   `data-alguma-coisa` correspondente.
2. Para cada seletor de styles.css, foram extraídas suas classes/IDs/
   atributos data-*. Um seletor só foi marcado como candidato a
   remoção se TODOS os seus tokens distintivos estavam ausentes do
   corpus (ou seja: nenhuma classe daquele seletor aparece em lugar
   nenhum do código, nem mesmo dentro de strings de template).
   Seletores de tag pura, universais (*), pseudo-classes soltas,
   :root, html, body e @keyframes/@font-face foram sempre preservados
   (risco alto demais para inferir automaticamente).
3. Cada bloco flagado foi checado manualmente por amostragem contra o
   HTML/JS atual antes da remoção (ex.: confirmado que
   ".avatar-compact-card" — com 261 ocorrências no CSS antigo — não
   aparece em nenhum lugar do index.html/js atual; o seletor de
   avatar em uso hoje é ".avatar-modal"/"#babyAvatarTestCard"/
   ".avatar-choice", de uma reformulação posterior do componente).

O QUE FOI REMOVIDO
-------------------
373 grupos de seletores (de um total de 3.641 analisados em
styles.css), somando ~2.300 linhas / ~65KB. A grande maioria pertence
a versões antigas e substituídas do seletor de avatar
(.avatar-compact-card e toda sua árvore de subseletores), além de
componentes pontuais (.wake-window-card, .active-timer-card antigo,
.guest-showcase-*, .field-chevron, etc.) que não são mais renderizados
pelo app atual.

Nenhum seletor de css/logged-premium.css foi removido (0 candidatos
encontrados nesse arquivo).

VERIFICAÇÃO
-----------
Um script independente recomputou o efeito de cascade
(contexto+seletor+propriedade) do arquivo final e comparou com o
arquivo anterior à remoção: das 12.996 combinações existentes antes,
11.594 permanecem — e são BYTE-A-BYTE IDÊNTICAS às de antes. As 1.402
combinações que sumiram pertencem exatamente aos 373 grupos removidos,
nenhuma outra.

RESSALVA IMPORTANTE
--------------------
Diferente da consolidação de duplicatas (v75.75.118), esta remoção
não é matematicamente à prova de falhas: ela depende da premissa de
que nenhuma classe é montada de forma totalmente dinâmica/computada
(ex.: `"avatar-" + variavel` sem a string completa aparecer em lugar
nenhum do código-fonte). Uma varredura manual no código não encontrou
nenhum padrão desse tipo, mas — como não é possível renderizar e
testar o app visualmente neste ambiente — recomenda-se um teste rápido
das telas de Perfil e do editor de avatar antes de publicar em
produção, para confirmar que nada que dependia desses seletores
"mortos" ainda está sendo usado de alguma forma não capturada pela
varredura.
